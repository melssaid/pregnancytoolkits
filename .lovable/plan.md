

# خطة تحسين تجربة ما بعد الـ Onboarding

## الوضع الحالي

بعد إكمال الـ Onboarding، يتم توجيه المستخدم إلى `/welcome-result` (شاشة النتيجة الذكية) التي تعرض:
- حالة الحمل والأسبوع
- 3 أدوات مقترحة
- توصية ذكية من AI
- زر واحد: "اذهب إلى لوحة التحكم"

المشكلة: المستخدم لا يرى الصفحة الرئيسية أبداً، ولا يحصل على خيار مباشر للوصول إليها.

---

## الاقتراح المبتكر: شاشة ترحيب ذكية مع توجيه مزدوج

بدلاً من توجيه المستخدم لمكان واحد فقط، نحوّل شاشة `/welcome-result` إلى **محطة انطلاق ذكية** مع تغييرين رئيسيين:

### 1. إضافة زرين بدل زر واحد

```text
┌─────────────────────────────────┐
│  ✨ رحلتك المخصصة جاهزة        │
│  [بطاقة حالة الحمل/الخصوبة]    │
│  [3 أدوات مقترحة]              │
│  [توصية AI]                     │
│                                 │
│  ┌─────────────────────────┐    │
│  │ 🏠 استكشف كل الأدوات   │ ← رئيسي → الصفحة الرئيسية
│  └─────────────────────────┘    │
│  ┌─────────────────────────┐    │
│  │ 📊 لوحة التحكم الخاصة  │ ← ثانوي → لوحة التحكم
│  └─────────────────────────┘    │
└─────────────────────────────────┘
```

- **الزر الأول (الرئيسي)**: يوجه للصفحة الرئيسية `/` لاستكشاف كل الأدوات
- **الزر الثاني (الثانوي)**: يوجه للوحة التحكم `/dashboard` للمتابعة الشخصية

### 2. تخصيص المحتوى حسب المرحلة

الشاشة تتكيف تلقائياً مع بيانات المستخدم:
- **حامل**: تعرض أسبوع الحمل + حجم الطفل + أدوات الحمل
- **خصوبة**: تعرض تتبع الدورة + المكملات + اللياقة
- **ما بعد الولادة**: تعرض الرضاعة + نوم الطفل + الصحة النفسية

(هذا موجود فعلاً ولن يتغير)

## الملخص

التغيير بسيط: نضيف زراً ثانياً في شاشة الترحيب بحيث يكون الزر الرئيسي يوجه للصفحة الرئيسية (كل الأدوات)، والزر الثانوي يوجه للوحة التحكم الشخصية. المستخدم يختار مساره بنفسه.
